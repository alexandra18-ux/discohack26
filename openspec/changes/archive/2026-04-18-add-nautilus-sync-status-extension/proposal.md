## Why

Сейчас пользователь может работать с Яндекс Диском через наш daemon, но в GNOME Files/Nautilus не видно, синхронизирован ли конкретный файл, находится ли он в очереди, загружается, скачивается или требует внимания. У daemon уже есть D-Bus API со статусами синхронизации и точкой монтирования, а также есть готовый референс интеграции Nautilus у Nextcloud, поэтому сейчас подходящий момент добавить нативные оверлеи статусов в файловый менеджер.

## What Changes

- Добавить расширение Nautilus для файловой схемы `file://`, которое определяет, относится ли файл или папка к смонтированному каталогу Яндекс Диска, и показывает эмблему статуса синхронизации.
- Подключить расширение к D-Bus сервису `ru.literallycats.daemon`, чтобы получать `MountPoint`, точечный статус через `GetSyncStatus(path)` и статусы содержимого каталога через `ListDirectoryStatuses(path)`.
- Обновлять отображение статусов при изменениях синхронизации без перезапуска Nautilus, используя существующие D-Bus обновления свойств и инвалидацию кэшированных элементов расширения.
- Добавить в проект код и установку расширения так, чтобы его можно было поставить вместе с приложением и использовать как штатную GNOME-интеграцию.
- Определить отображение daemon-состояний (`synced`, `queued`, `uploading`, `downloading`, `conflict`, `error`, `unknown`) в набор доступных эмблем и безопасное поведение при недоступности D-Bus сервиса.

## Capabilities

### New Capabilities
- `nautilus-sync-status-extension`: GNOME Files/Nautilus показывает статусы синхронизации Яндекс Диска для файлов и папок через D-Bus интеграцию с daemon.

### Modified Capabilities
- None.

## Impact

- Будут затронуты структура проекта и сборка/установка: появится код Nautilus extension и шаг установки в каталог расширений `nautilus-python`.
- Новая интеграция будет зависеть от Python + `nautilus-python` + PyGObject/GIO D-Bus API в рантайме.
- Расширение будет использовать существующий D-Bus контракт daemon в `../discohack-daemon/src/dbus_service.rs` и полагаться на `MountPoint`, `GetSyncStatus`, `ListDirectoryStatuses` и обновления `PropertiesChanged`.
- Потребуется сопоставление локальных путей Nautilus с путями внутри смонтированного дерева Яндекс Диска и локальный кэш статусов внутри процесса расширения.
