## Why

Файл `src/main.vala` был переименован в `src/statewidget.vala`, но в нем по-прежнему смешаны сразу несколько ролей: сам переиспользуемый виджет состояния, окно приложения и точка входа. Из-за этого компонент неудобно использовать модульно в собственной реализации нового `main.vala`, а сборка остается привязанной к старой структуре исходников.

## What Changes

- Разделить код виджета состояния и код приложения по отдельным исходным файлам с понятными зонами ответственности.
- Оставить `statewidget.vala` как модуль с `StatusState` и `StatusStateWidget`, пригодный для подключения из другого файла.
- Создать/обновить `main.vala`, чтобы он использовал модульный виджет через явный API без дублирования логики компонента.
- Обновить `src/meson.build`, чтобы сборка включала оба исходных файла и больше не зависела от того, что весь код находится в одном файле.

## Capabilities

### New Capabilities
- `modular-status-widget`: Приложение предоставляет виджет состояния как отдельный Vala-модуль, который можно подключать и использовать из самостоятельной реализации `main.vala`.

### Modified Capabilities
- `status-state-widget`: Интеграция виджета в основное окно должна работать после разделения кода по модулям и отдельным файлам.

## Impact

- Будут затронуты `src/statewidget.vala`, новый или обновленный `src/main.vala` и `src/meson.build`.
- Изменится только структура исходников и точки интеграции в UI-слое без новых внешних зависимостей.
- Сборка Meson будет явно собирать модуль виджета и файл точки входа приложения.
